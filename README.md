
# Многослойный перцептрон для классификации MNIST: полное теоретическое обоснование

## Введение

Многослойный перцептрон (Multilayer Perceptron, MLP) представляет собой класс искусственных нейронных сетей прямого распространения, состоящий из нескольких слоев искусственных нейронов. Это один из фундаментальных алгоритмов машинного обучения, который демонстрирует способность к аппроксимации любых непрерывных функций при достаточном количестве скрытых нейронов.

## 1. Архитектура многослойного перцептрона

### 1.1 Общая структура

Наша архитектура представляет собой полносвязную нейронную сеть следующей конфигурации:

```
Входной слой: 784 нейрона (28×28 пикселей изображения)
    ↓ (Leaky ReLU)
Скрытый слой 1: 256 нейронов
    ↓ (Leaky ReLU)  
Скрытый слой 2: 256 нейронов
    ↓ (Sigmoid)
Выходной слой: 10 нейронов (классы цифр 0-9)
```

### 1.2 Математическое представление

Каждый слой можно описать линейным преобразованием с последующим применением функции активации:

**Слой 1:**

```
z₁ = X · W₁ + b₁
a₁ = LeakyReLU(z₁)
```

**Слой 2:**

```
z₂ = a₁ · W₂ + b₂  
a₂ = LeakyReLU(z₂)
```

**Выходной слой:**

```
z₃ = a₂ · W₃ + b₃
a₃ = Sigmoid(z₃)
```

Где:

- `W` — матрицы весов
- `b` — векторы смещений (bias)
- `z` — взвешенные суммы (логиты)
- `a` — активации после функции активации

## 2. Функции активации

### 2.1 Leaky ReLU

Функция Leaky ReLU решает проблему "умирающих нейронов" классической ReLU, позволяя небольшому градиенту проходить через отрицательные значения:

```python
f(x) = max(αx, x), где α = 0.01
```

**Производная:**

```python
f'(x) = {1,  если x > 0
        {α,  если x ≤ 0
```

**Преимущества:**

- Предотвращает проблему исчезающего градиента
- Вычислительно эффективна
- Не насыщается для положительных значений

### 2.2 Sigmoid

Сигмоидная функция используется в выходном слое для получения вероятностей принадлежности к классам:

```python
f(x) = 1/(1 + e^(-x))
```

**Производная:**

```python
f'(x) = f(x) × (1 - f(x))
```

**Особенности:**

- Выход в диапазоне (0, 1)
- Подходит для многоклассовой классификации
- Может приводить к проблеме исчезающего градиента в глубоких сетях

## 3. Прямое распространение (Forward Propagation)

Прямое распространение — это процесс вычисления выхода сети для данного входа. Алгоритм последовательно применяет линейные преобразования и функции активации:

### 3.1 Пошаговый алгоритм

1. **Входной слой:** Нормализованные пиксели изображения `X ∈ ℝ^(m×784)`
2. **Первый скрытый слой:**
    - Линейная комбинация: `z₁ = X·W₁ + b₁`
    - Активация: `a₁ = LeakyReLU(z₁)`
3. **Второй скрытый слой:**
    - Линейная комбинация: `z₂ = a₁·W₂ + b₂`
    - Активация: `a₂ = LeakyReLU(z₂)`
4. **Выходной слой:**
    - Линейная комбинация: `z₃ = a₂·W₃ + b₃`
    - Активация: `ŷ = Sigmoid(z₃)`

### 3.2 Размерности тензоров

Для батча размером `m`:

- `X: (m, 784)`
- `W₁: (784, 256)`, `b₁: (1, 256)`, `a₁: (m, 256)`
- `W₂: (256, 256)`, `b₂: (1, 256)`, `a₂: (m, 256)`
- `W₃: (256, 10)`, `b₃: (1, 10)`, `ŷ: (m, 10)`

## 4. Функция потерь

Для многоклассовой классификации используется кроссэнтропийная функция потерь:

```python
L = -1/m × Σᵢ₌₁ᵐ Σⱼ₌₁ᶜ yᵢⱼ × log(ŷᵢⱼ)
```

Где:

- `m` — размер батча
- `c` — количество классов (10)
- `yᵢⱼ` — истинная метка (one-hot encoded)
- `ŷᵢⱼ` — предсказанная вероятность

## 5. Обратное распространение ошибки

Обратное распространение ошибки — это алгоритм вычисления градиентов функции потерь по параметрам сети. Основан на применении цепного правила дифференцирования.

### 5.1 Цепное правило

Для композиции функций `f(g(x))` производная вычисляется как:

```
∂f(g(x))/∂x = ∂f/∂g × ∂g/∂x
```

### 5.2 Вычисление градиентов для выходного слоя

**Градиент по выходу сети:**

```python
∂L/∂a₃ = -(y - a₃)/m
```

Поскольку используется sigmoid в качестве активации выходного слоя:

```python
∂L/∂z₃ = ∂L/∂a₃ × ∂a₃/∂z₃ = (a₃ - y)/m
```

**Градиенты по параметрам выходного слоя:**

```python
∂L/∂W₃ = a₂ᵀ × ∂L/∂z₃
∂L/∂b₃ = Σ(∂L/∂z₃)  # сумма по батчу
```

### 5.3 Вычисление градиентов для скрытых слоев

**Второй скрытый слой:**

Ошибка распространяется назад через веса:

```python
∂L/∂a₂ = ∂L/∂z₃ × W₃ᵀ
```

Применяем производную Leaky ReLU:

```python
∂L/∂z₂ = ∂L/∂a₂ × ∂a₂/∂z₂ = ∂L/∂a₂ × LeakyReLU'(z₂)
```

Градиенты по параметрам:

```python
∂L/∂W₂ = a₁ᵀ × ∂L/∂z₂
∂L/∂b₂ = Σ(∂L/∂z₂)
```

**Первый скрытый слой:**

Аналогично:

```python
∂L/∂a₁ = ∂L/∂z₂ × W₂ᵀ
∂L/∂z₁ = ∂L/∂a₁ × LeakyReLU'(z₁)
∂L/∂W₁ = Xᵀ × ∂L/∂z₁
∂L/∂b₁ = Σ(∂L/∂z₁)
```

### 5.4 Полный алгоритм обратного распространения

```python
def backward_propagation(X, y, cache):
    m = X.shape[0]
    
    # Распаковка кеша прямого прохода
    z1, a1, z2, a2, z3, a3 = cache
    
    # Выходной слой
    dz3 = (a3 - y) / m
    dW3 = np.dot(a2.T, dz3)
    db3 = np.sum(dz3, axis=0, keepdims=True)
    
    # Второй скрытый слой
    da2 = np.dot(dz3, W3.T)
    dz2 = da2 * leaky_relu_derivative(z2)
    dW2 = np.dot(a1.T, dz2)
    db2 = np.sum(dz2, axis=0, keepdims=True)
    
    # Первый скрытый слой
    da1 = np.dot(dz2, W2.T)
    dz1 = da1 * leaky_relu_derivative(z1)
    dW1 = np.dot(X.T, dz1)
    db1 = np.sum(dz1, axis=0, keepdims=True)
    
    return dW1, db1, dW2, db2, dW3, db3
```

## 6. Градиентный спуск

После вычисления градиентов параметры обновляются согласно правилу градиентного спуска:

```python
W = W - α × ∂L/∂W
b = b - α × ∂L/∂b
```

Где `α` — скорость обучения (learning rate).

### 6.1 Мини-батч градиентный спуск

Для улучшения сходимости и эффективности используется мини-батч градиентный спуск:

1. Разбиваем тренировочный набор на батчи размера `batch_size`
2. Для каждого батча вычисляем градиенты
3. Обновляем параметры на основе усредненных градиентов

**Преимущества:**

- Более стабильная сходимость по сравнению со стохастическим градиентным спуском
- Эффективное использование векторизованных операций
- Меньше шума в градиентах по сравнению с SGD

## 7. Инициализация весов

Правильная инициализация весов критически важна для успешного обучения.

### 7.1 Xavier/Glorot инициализация

Для слоев с сигмоидными функциями активации:

```python
W ~ N(0, √(2/(n_in + n_out)))
```

### 7.2 He инициализация

Для слоев с ReLU-активациями:

```python
W ~ N(0, √(2/n_in))
```

Где `n_in` — количество входящих связей.

**Обоснование:** Поддерживает дисперсию активаций и градиентов примерно постоянной на всех слоях, предотвращая проблемы исчезающего/взрывающегося градиента.

## 8. Датасет MNIST

### 8.1 Характеристики

- **Размер:** 70,000 изображений (60,000 тренировочных, 10,000 тестовых)
- **Разрешение:** 28×28 пикселей, градации серого
- **Классы:** Цифры от 0 до 9
- **Формат:** Значения пикселей в диапазоне [0, 255]

### 8.2 Предобработка

1. **Нормализация:** Пиксели приводятся к диапазону [0, 1] делением на 255
2. **Векторизация:** Изображения 28×28 преобразуются в векторы размерности 784
3. **One-hot encoding:** Метки классов преобразуются в векторы длины 10

```python
# Нормализация
X = X.astype('float32') / 255.0

# Векторизация  
X = X.reshape(X.shape[0], -1)  # (n_samples, 784)

# One-hot encoding
y_onehot = np.zeros((n_samples, 10))
y_onehot[np.arange(n_samples), y] = 1
```

## 9. Метрики качества

### 9.1 Точность (Accuracy)

```python
accuracy = (число правильных предсказаний) / (общее число предсказаний)
```

### 9.2 Функция потерь

Кроссэнтропийная потеря показывает, насколько хорошо предсказанные вероятности соответствуют истинным меткам.

## 10. Проблемы и решения

### 10.1 Исчезающий градиент

**Проблема:** Градиенты становятся экспоненциально малыми при распространении через глубокие слои.

**Решения:**

- Использование Leaky ReLU вместо sigmoid в скрытых слоях
- Правильная инициализация весов
- Нормализация батчей (в более сложных архитектурах)

### 10.2 Переобучение

**Проблема:** Модель запоминает тренировочные данные, плохо обобщает на новые.

**Решения:**

- Регуляризация (L1/L2)
- Dropout
- Валидационный контроль
- Аугментация данных

### 10.3 Локальные минимумы

**Проблема:** Градиентный спуск может застрять в локальных минимумах.

**Решения:**

- Случайная инициализация
- Различные скорости обучения
- Momentum и другие оптимизаторы

## 11. Вычислительная сложность

### 11.1 Прямое распространение

- **Сложность:** O(n × m), где n — количество параметров, m — размер батча
- **Операции:** Матричные умножения, поэлементные операции

### 11.2 Обратное распространение

- **Сложность:** O(n × m) — такая же, как прямое распространение
- **Операции:** Вычисление градиентов через цепное правило

Многослойный перцептрон представляет собой мощный инструмент для решения задач классификации. Ключевые моменты успешной реализации:

1. **Архитектура:** Достаточное количество скрытых нейронов для изучения сложных зависимостей
2. **Активации:** Правильный выбор функций активации для предотвращения проблем с градиентами
3. **Инициализация:** Правильная инициализация весов для стабильного обучения
4. **Оптимизация:** Эффективная реализация алгоритма обратного распространения ошибки
5. **Регуляризация:** Методы предотвращения переобучения

Алгоритм обратного распространения ошибки является краеугольным камнем обучения нейронных сетей, позволяя эффективно вычислять градиенты сложных композитных функций и обновлять параметры модели для минимизации функции потерь.
